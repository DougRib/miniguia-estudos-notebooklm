# Checklist de entrega — DIO

Use este checklist antes de enviar o projeto na plataforma da DIO.

## GitHub

- [ ] Repositório criado no GitHub.
- [ ] Repositório está público.
- [ ] Nome do repositório é claro e descritivo.
- [ ] README.md está na raiz do projeto.
- [ ] README contém contexto e objetivos.
- [ ] README contém fontes utilizadas.
- [ ] README contém prompts testados.
- [ ] README contém cicatrizes do processo.
- [ ] README contém miniguia final.
- [ ] README contém glossário.
- [ ] README contém conclusão.

## NotebookLM

- [ ] Caderno criado no NotebookLM.
- [ ] Tema definido.
- [ ] Fontes adicionadas.
- [ ] Prompts testados.
- [ ] Respostas analisadas.
- [ ] Miniguia gerado.
- [ ] Glossário criado.
- [ ] Prints salvos na pasta `imagens/`.

## Arquivos complementares

- [ ] `fontes/fontes-utilizadas.md`

- [ ] `guia/miniguia-estudo.md`

## Entrega na DIO

- [ ] Copiar a URL principal do repositório.
- [ ] Acessar a página do desafio.
- [ ] Clicar em entregar projeto.
- [ ] Colar o link do GitHub.
- [ ] Adicionar uma descrição objetiva.
- [ ] Confirmar a entrega.

## Descrição sugerida para a DIO

```text
Projeto desenvolvido com NotebookLM como ferramenta de aprendizagem ativa. O repositório apresenta um segundo cérebro de estudos sobre Yann LeCun, com curadoria de fontes, prompts testados, registro de dificuldades, glossário e miniguia sobre suas contribuições para Inteligência Artificial, deep learning, CNNs, LeNet, JEPA e world models.
```
